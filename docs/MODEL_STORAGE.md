# Model Storage — Beavis Agent

## 1. Main rule

Generated model files must not be committed to Git.

This includes:

```text
*.joblib
STT model files
downloaded faster-whisper models
large binary model artifacts
```

The repository should keep only source code, configs, datasets/scripts, and placeholder files.

## 2. Correct `.gitignore` block

Use this block in `.gitignore`:

```gitignore
# ML / generated models
python_agent/models/*
!python_agent/models/.gitkeep
!python_agent/models/stt/
python_agent/models/stt/*
!python_agent/models/stt/.gitkeep
```

This means:

```text
ignore everything inside python_agent/models/
keep python_agent/models/.gitkeep
keep the stt folder itself
ignore everything inside python_agent/models/stt/
keep python_agent/models/stt/.gitkeep
```

## 3. Correct folder structure

The repository should contain:

```text
python_agent/models/.gitkeep
python_agent/models/stt/.gitkeep
```

The repository should not contain:

```text
python_agent/models/*.joblib
python_agent/models/stt/*
```

## 4. Why models are ignored

Reasons:

```text
GitHub blocks normal Git files over 100 MiB.
Files over 50 MiB produce warnings.
STT models can be very large.
Generated joblib models are build artifacts.
Joblib models may depend on exact sklearn/joblib versions.
Local models can be regenerated by training scripts.
```

## 5. Remove models from current Git tracking

From repository root:

```powershell
cd C:\Users\gerychhh_\beavis_agent
```

Create placeholders:

```powershell
New-Item -ItemType File -Force python_agent/models/.gitkeep
New-Item -ItemType File -Force python_agent/models/stt/.gitkeep
```

Remove models from Git index but keep them on disk:

```powershell
git rm -r --cached python_agent/models/
```

Add only ignore rules and placeholders:

```powershell
git add .gitignore
git add python_agent/models/.gitkeep
git add python_agent/models/stt/.gitkeep
```

Commit:

```powershell
git commit -m "Remove generated model files from repository"
git push
```

Check:

```powershell
git ls-files python_agent/models/
```

Expected output:

```text
python_agent/models/.gitkeep
python_agent/models/stt/.gitkeep
```

## 6. If models were added in the first commit

If old model files were added in the first commit, a normal removal commit deletes them only from the latest version.

They still remain in Git history.

If you need to completely remove them from history, use a clean orphan history.

### Step 1 — create backup branch

```powershell
git branch backup-old-history
```

### Step 2 — create placeholders

```powershell
New-Item -ItemType File -Force python_agent/models/.gitkeep
New-Item -ItemType File -Force python_agent/models/stt/.gitkeep
```

### Step 3 — create clean orphan branch

```powershell
git checkout --orphan clean-master
```

### Step 4 — clear Git index

```powershell
git rm -r --cached .
```

### Step 5 — add files again

```powershell
git add .
```

Because `.gitignore` ignores generated models, model binaries should not be added.

Check before commit:

```powershell
git ls-files python_agent/models/
```

Expected:

```text
python_agent/models/.gitkeep
python_agent/models/stt/.gitkeep
```

### Step 6 — create new clean commit

```powershell
git commit -m "Initial clean project"
```

### Step 7 — replace master locally

```powershell
git branch -D master
git branch -m master
```

### Step 8 — force push

```powershell
git push --force origin master
```

After this, GitHub will show a clean history.

Important:

```text
Force push rewrites remote history.
Old local clones should be deleted and cloned again.
```

## 7. Training local models

After cloning the repository, generate local models:

```powershell
.\scripts\dev.ps1 train
```

This should generate models such as:

```text
python_agent/models/skill_classifier.joblib
python_agent/models/window_control_arg_model.joblib
python_agent/models/window_layout_arg_model.joblib
```

Other model training scripts may generate:

```text
python_agent/models/open_app_arg_model.joblib
python_agent/models/volume_set_arg_model.joblib
```

Generated files stay local and ignored.

## 8. Checking tracked model files

Show tracked files under models:

```powershell
git ls-files python_agent/models/
```

Good output:

```text
python_agent/models/.gitkeep
python_agent/models/stt/.gitkeep
```

Bad output:

```text
python_agent/models/skill_classifier.joblib
python_agent/models/open_app_arg_model.joblib
python_agent/models/volume_set_arg_model.joblib
python_agent/models/window_control_arg_model.joblib
python_agent/models/window_layout_arg_model.joblib
```

If bad output appears, remove tracked models:

```powershell
git rm -r --cached python_agent/models/
git add python_agent/models/.gitkeep
git add python_agent/models/stt/.gitkeep
git commit -m "Stop tracking generated models"
```

## 9. Checking large files before push

PowerShell command:

```powershell
git ls-files | ForEach-Object {
    if (Test-Path $_) {
        $size = (Get-Item $_).Length
        [PSCustomObject]@{
            SizeMB = [math]::Round($size / 1MB, 2)
            Path = $_
        }
    }
} | Sort-Object SizeMB -Descending | Select-Object -First 20
```

Files near or above 50 MiB should not be committed unless there is a specific reason.

## 10. Recommended policy

Use this policy:

```text
Do not commit generated models.
Do not commit downloaded STT models.
Do not commit runtime logs.
Do not commit local app indexes.
Do not commit user app catalogs.
Do commit training scripts.
Do commit dataset generation scripts.
Do commit small config files.
Do commit docs.
Do commit .gitkeep placeholders.
```
